# 比巴卜作手视频分析

## 执行前必读

处理视频前读取 [references/video-analysis-workflow.md](references/video-analysis-workflow.md)，按“元数据核验 → 字幕或转写 → 观点时间轴 → 关键帧 → 名词核验 → 文档更新 → 构建校验”的顺序执行。

## 工作流

1. 核验视频标题、UP 主、发布日期、时长和原始链接。账号名以 B 站页面为准。
2. 获取字幕；无字幕时转写音频。文字稿必须分两步制作：先形成忠实保留识别内容的原始提取稿，再以该原稿为底稿划分段落、轻度润色和修订明显错误。完成覆盖全片的可读时间轴文字稿后，才进行个股核名和总结分析；不确定的个股名标记待核，不得静默遗漏。
3. 按以下顺序梳理内容：
   - 指数环境：反弹或反转、量能、压力位、次日确认条件。
   - 赚钱效应：涨停、封板、连板及资金所在风格。
   - 具体分析：逐个板块记录制作者看法、强弱排序、催化因素和次日关注。
   - 全部个股：逐板块列出视频明确提到的每一只股票，再说明其中的核心、中军、辨识度、竞争关系和失效条件。
   - 交易计划：买点前提、仓位、卖点、次日预案。
   - 心态纪律：主播明确表达的风险控制原则。
   - 篇幅较长的观点：无论属于上述哪个部分，都要按作者原有展开顺序充分保留论据、推演、例子、转折、条件和结论；必要时在对应大模块下单设多个小点，不得压缩成一句结论或仅保留摘要。
4. 选择足以覆盖市场总览和主要板块的关键截图，通常为 6 至 10 张。优先截取图表、计划表、板块对比、重点个股和失效条件，避免只截口播画面。
5. 每张图注明原视频时间点。解释优先沿用作者的表达顺序、判断措辞和语气，再补充必要的结构化说明；截图仅作学习引用，不替代原视频。
6. 每日总结固定分为“每日信息”和“知识点”两个一级部分。知识点中的延展内容必须说明是分析者推导，不冒充主播原话。
7. 输出到人物专题时，采用“人物主页 → 技术点学习 / 交易日总结 → 日期文章”的目录层级；只有交易日总结按日期持续新增文章。
8. “每日信息”固定包含三个部分：
   - 第一部分“市场总览”：指数、赚钱效应、资金结构和一句话结论。
   - 第二部分“具体分析”：先给出板块观点速览表，再逐板块整理制作者观点、重点个股、截图依据和次日关注。
   - 第三部分“次日计划”：指数预案、板块预案、仓位动作、失效条件和观察清单。
9. “知识点”记录当日可迁移方法、延展知识、适用条件和失效条件。
10. 独立的“技术点学习”模块用于积累多个交易日后做跨期归纳。除非用户明确要求归纳或更新，否则只保留模块入口，不新增日期文章、不搬运单日知识点。
11. 每期必须在本项目中产出两个文件，固定名称为“交易日总结”和“视频文字稿”，不得使用其他模块名：
   - `docs/trading/experts/bibabu-zuoshou/transcripts/YYYY-MM-DD.md`：覆盖全片议题的可读时间轴文字稿。
   - `docs/trading/experts/bibabu-zuoshou/trading-days/YYYY-MM-DD.md`：基于完整文字稿制作的图文分析总结。
12. 保留原视频链接、项目内文字稿链接与“非投资建议”声明，并同步更新人物主页、交易日总结索引、文字稿索引和 VitePress 侧边栏。
13. 确认项目文字稿、总结和关键帧全部落盘后，逐个删除本期在 `/tmp` 生成的视频、音频、原始自动转写、候选帧和联系表，不保留本地临时副本。

## 工具选择原则

- 首选 B 站公开接口读取标题、UP 主、发布时间、时长、`cid`、字幕列表和播放地址。
- 首选视频自带字幕；字幕缺失或质量不足时使用 `faster-whisper` 转写。
- 使用 PyAV 精确定位时间点，Pillow 输出 JPEG 和候选帧联系表，再人工复核图片。
- 使用 `rg`、`sed` 检索转写稿，建立观点时间轴和全部股票提及清单，再组织文字稿与总结。
- 生僻股票名和题材名按“清晰画面 > 官方元数据或字幕 > 上下文 > 语音识别”核验。
- 使用 `quick_validate.py`、站点构建命令和 `git diff --check` 做发布前检查。
- 原视频、音频和未经整理的自动转写只允许暂存在临时目录，不提交到仓库。整理后的可读时间轴文字稿必须写入本项目，任务结束前逐个清除本期全部临时文件。

## 文字稿规范

- 文字稿制作分为两个连续步骤，不得跳过原始提取稿直接写结构化文字稿或总结：
  1. **原始提取稿**：直接落下字幕或语音识别结果，尽量保持作者原始顺序、说法、语气强弱、重复、犹豫、自我修正、类比和例子，不先概括、改写、重组、核名替换或补充分析。仅可补充时间点、说话人和无法识别标记。
  2. **整理文字稿**：必须在原始提取稿上逐段处理，划分自然段和议题，做不改变原意与语气强弱的轻度文字润色，删除纯填充语气词和无意义重复，并结合画面、上下文及可靠信息修订明显识别错误、核对股票和题材名称。
- 原始提取稿是过程性底稿，可暂存在临时目录，不要求提交；项目中的视频文字稿为第二步完成的整理稿。
- 覆盖从开场到收尾的全部有效议题，不只摘录高光、结论或代表性股票。
- 每个自然议题标注起始时间，行情切换、板块切换、个股切换和观点转折处必须增加时间点。
- 整理时不得把口语改写成摘要或分析结论；最大化保留作者的论述顺序、常用说法、语气强弱、犹豫、自我修正、类比、例子和有信息量的重复，不能确定是否有信息量时予以保留。
- 作者对某一观点讲述较多、反复补充或连续推演时，整理文字稿必须给予与原视频相称的篇幅，完整保留其问题背景、判断依据、推理过程、举例、反例、条件限制、观点转折和最终落点。不得因观点可以概括而大幅缩写，也不得只留下首句和结论。
- 长观点可在同一时间轴议题下按作者的展开层次拆成多个自然段或小点，以提高可读性；拆分只能整理结构，不能合并、重排或删减关键论述。
- 自动转写不能直接标成逐字原话。明显识别错误按画面和上下文修正，无法确认的股票或专有名词保留时间点并标记“待核”。
- 单独维护“本期全部股票提及表”，记录股票名、代码（能确认时）、所属板块、出现时间、作者说法和核验状态。
- 同一股票多次出现时，不只保留第一次；应合并列出关键时间点和观点变化。
- 开头注明视频信息、转写来源、视频时长和准确性限制，并链接回原视频。
- 后续核名、截图选择和总结必须以整理文字稿为依据；不得在整理文字稿尚未完成时直接根据零散识别片段制作总结。

## 分析准则

- 将预测改写为条件句，例如“若次日放量站稳压力位，反转概率才提高”。
- 区分板块“涨得多”和“有持续性”。持续性至少考察逆势抗跌、修复主动、核心带动、量价健康。
- 区分消息刺激和内生趋势。消息日的一致高潮不等于可追买。
- 记录否证条件。没有退出条件的看多观点不算完整交易计划。
- 对个股仅复述视频语境，不提供确定性买卖指令。
- 板块速览表至少包含“板块、制作者观点、全部提及个股、次日关注”四列。
- 视频明确提及的股票必须全部进入文字稿和“全部股票提及表”，不能因为不是核心、只出现一次或观点偏负面而省略。
- 总结正文也要按板块保留全部提及股票；可另行标记核心、跟风、中军、竞争者、持仓、观察或回避，但不能用“代表性个股”替代完整记录。
- 股票名无法确认时保留其时间点、上下文和“待核”标记，不凭发音猜测，也不因不确定而静默删除。
- 总结优先忠实呈现作者的原始表达逻辑。结构化改写不得改变语气强弱，例如不能把“感觉可以看看”改成“明确看好”。
- 总结的详略应参考作者在视频中的实际论述篇幅和强调程度。作者展开较多的观点必须在总结中充分保留其依据、推演链条、例子、条件、风险和结论，不得为了控制篇幅压缩成速览表中的一句话或一条笼统结论。
- 对篇幅较长或层次较多的观点，应在“市场总览”“具体分析”“次日计划”或“知识点”等对应大模块中单独设置清晰的小点；即使速览表已经概括，也必须在正文继续展开。一个长观点涉及多个不同判断时，可拆成多个小点逐一保留。
- 对每个主要板块说明其定位：主线候选、消息修复、高低切、量化轮动、情绪 PK、避险或回避。
- 如转写与画面冲突，以画面和上下文为准；仍不确定时使用“疑似”或省略名称。

### 人物主页

只维护两个子模块的入口和说明，不堆放日期归档或单期正文。

### 交易日总结模块

在 `trading-days/index.md` 按日期倒序维护文章入口。每篇日期文章固定包含：

- `## 每日信息`
  - `### 第一部分：市场总览`
  - `### 第二部分：具体分析`
  - `### 第三部分：次日计划`
- `## 知识点`：当日方法提炼、适用条件、常见误区、失效条件和交易准则，明确标注分析与延展。

固定一级、二级结构不限制正文展开。凡作者在视频中重点长篇讨论的内容，可在对应部分下增加四级标题、加粗小标题或多个独立小点，篇幅以完整保留观点为优先，不以追求短摘要为目标。

### 文字稿模块

在本项目的 `docs/trading/experts/bibabu-zuoshou/transcripts/` 保存文字稿，并在 `transcripts/index.md` 按日期倒序维护入口。每篇文字稿包含视频信息、准确性说明、全片时间轴和“本期全部股票提及表”。文字稿服务于追溯作者完整表达，不替代原视频，也不把自动识别文本冒充逐字稿。

`trading-days/index.md` 的模块标题固定为“交易日总结”，`transcripts/index.md` 的模块标题固定为“视频文字稿”；日期文章标题分别使用 `YYYY-MM-DD 交易日总结` 和 `YYYY-MM-DD 视频文字稿`。

### 技术点学习模块

`technical-notes/index.md` 是未来跨期归纳入口。默认不从单日视频直接更新；积累足够样本并经用户明确要求后，再按主题总结稳定方法、常见偏差及适用边界。

截图统一保存到 `docs/public/images/trading/bibabu/YYYY-MM-DD/`，文件名优先使用四位秒数，便于追溯原视频时间点。

## 专属观察框架

读取 [references/expert-profile.md](references/expert-profile.md)，持续维护其稳定方法、常用指标、题材分类方式和历史观点命中情况。每期只在有新证据时更新画像，不因单条视频下结论。
