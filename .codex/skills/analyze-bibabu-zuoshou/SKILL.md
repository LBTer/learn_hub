# 比巴卜作手视频分析

## 执行前必读

处理视频前读取 [references/video-analysis-workflow.md](references/video-analysis-workflow.md)，按“元数据确认 → 文字稿提取与整理 → 观点时间轴 → 文字稿阶段核实全部股票名 → 文字稿冻结 → 总结生成 → 关键帧与文档更新 → 构建校验”的顺序执行。

## 类型识别与路由

1. 获取标题、UP 主、发布日期、时长、原始链接、分 P、`cid`、字幕列表、页面可见性和充电限制信息。
2. 按以下证据识别类型，证据冲突时以 B 站页面可见性和实际内容结构为准：
   - **普通复盘视频**：公开视频，围绕当日指数、题材、个股、交易计划和次日应对展开。
   - **充电会员复盘视频**：页面或接口显示充电专属、会员可见、合集属于复盘，内容仍以市场复盘、板块个股、交易应对和新手教学为主。
   - **充电会员专题视频**：页面或接口显示充电专属、会员可见，但内容不是单日复盘，而是围绕交易方法、情绪周期、打板教学、案例课、心态纪律或其他专题展开。
3. 充电会员视频按内容继续路由：
   - **充电会员复盘视频**读取 [references/member-video.md](references/member-video.md)，生成会员复盘文字稿和会员复盘笔记。
   - **充电会员专题课程**读取 [references/member-course-video.md](references/member-course-video.md)，生成课程文字稿和课程学习笔记，并维护所属合集的体系地图。
   - 若本期实际框架暴露新栏目、新合集或新分析维度，优先在本期文档中如实处理，再回头迭代对应工作流。
4. 普通复盘视频继续执行下方“普通复盘工作流”。
5. 不得把充电会员复盘硬套成普通每日总结，也不得把普通公开视频写入会员目录。

## 普通复盘工作流

1. 核验视频标题、UP 主、发布日期、时长和原始链接。账号名以 B 站页面为准。
2. 必须严格串行执行以下阶段，不得交叉生成文字稿和总结：
   - **阶段一：文字稿提取与整理**。获取字幕；无字幕时转写音频。先形成忠实保留识别内容的原始提取稿，再以原稿为底稿制作覆盖全片的可读时间轴文字稿。此阶段不写总结。
   - **阶段二：股票名核实**。以整理文字稿和“全部股票提及清单”为依据，结合清晰画面、字幕、上下文及必要的公开行情信息，逐一核实视频提到的全部股票名称；无法确认时保留时间点和上下文并标记“待核”。
   - **阶段门：文字稿冻结**。确认全片有效议题、全部股票、新手教学、长篇观点和次日计划均已纳入，且“本期全部股票提及表”已完成。将文字稿完整写入正式文件后冻结。
   - **阶段三：总结生成**。只能在文字稿冻结后创建或填写交易日总结，并以冻结文字稿为唯一内容底稿。
   - **阶段四：发布检查**。完成关键帧、交叉检查、索引与侧边栏更新、构建校验和临时文件清理。
3. 内容核实仅限于文字稿阶段的股票名称核实。除视频元数据确认外，不要求对题材逻辑、市场数据、主播判断或新闻背景做额外事实核验；不得把外部信息补写成主播观点。总结阶段不再启动独立核实，只忠实整理冻结文字稿中的内容和已确定的股票名。
4. 在文字稿冻结前禁止创建或预写总结正文。总结不得直接基于零散字幕、音频片段、原始提取稿、标题、截图或记忆生成；冻结后若文字稿发生实质修订，必须重新检查并同步修订总结。
5. 按以下顺序梳理内容：
   - 指数环境：反弹或反转、量能、压力位、次日确认条件。
   - 赚钱效应：涨停、封板、连板及资金所在风格。
   - 具体分析：逐个板块记录制作者看法、强弱排序、催化因素和次日关注。
   - 全部个股：逐板块列出视频明确提到的每一只股票，再说明其中的核心、中军、辨识度、竞争关系和失效条件。
   - 交易计划：买点前提、仓位、卖点、次日预案。
   - 心态纪律：主播明确表达的风险控制原则。
   - 篇幅较长的观点：无论属于上述哪个部分，都要按作者原有展开顺序充分保留论据、推演、例子、转折、条件和结论；必要时在对应大模块下单设多个小点，不得压缩成一句结论或仅保留摘要。
6. 选择足以覆盖市场总览和主要板块的关键截图，通常为 6 至 10 张。优先截取图表、计划表、板块对比、重点个股和失效条件，避免只截口播画面。
7. 每张图注明原视频时间点。解释优先沿用作者的表达顺序、判断措辞和语气，再补充必要的结构化说明；截图仅作学习引用，不替代原视频。
8. 每日总结固定分为“每日信息”和“知识点”两个一级部分。知识点中的延展内容必须说明是分析者推导，不冒充主播原话。
9. 输出到人物专题时，采用“人物主页 → 技术点学习 / 交易日总结 / 视频文字稿 → 日期文章”的目录层级；普通复盘只写入普通复盘目录，会员视频写入 `member-topics/` 下的对应子目录。
10. “每日信息”固定包含三个部分：
   - 第一部分“市场总览”：指数、赚钱效应、资金结构和一句话结论。
   - 第二部分“具体分析”：先给出板块观点速览表，再逐板块整理制作者观点、重点个股、截图依据和次日关注。
   - 第三部分“次日计划”：指数预案、板块预案、仓位动作、失效条件和观察清单。
11. “知识点”记录当日可迁移方法、延展知识、适用条件和失效条件。
12. 独立的“技术点学习”模块用于积累多个交易日后做跨期归纳。除非用户明确要求归纳或更新，否则只保留模块入口，不新增日期文章、不搬运单日知识点。
13. 每期普通复盘必须在本项目中产出两个文件，固定名称为“交易日总结”和“视频文字稿”，不得使用其他模块名：
   - `docs/trading/experts/bibabu-zuoshou/transcripts/YYYY-MM-DD.md`：覆盖全片议题的可读时间轴文字稿。
   - `docs/trading/experts/bibabu-zuoshou/trading-days/YYYY-MM-DD.md`：基于完整文字稿制作的图文分析总结。
14. 保留原视频链接、项目内文字稿链接与“非投资建议”声明，并同步更新人物主页、交易日总结索引、文字稿索引和 VitePress 侧边栏。每新增、改名或迁移一期文章，都必须检查 `docs/.vitepress/config.mts` 中“比巴卜作手”目录，按日期倒序新增或修正对应的交易日总结、视频文字稿或会员文章入口；不得因模块索引已更新而跳过侧边栏目录更新。
15. 确认项目文字稿、总结和关键帧全部落盘后，逐个删除本期在 `/tmp` 生成的视频、音频、原始自动转写、候选帧和联系表，不保留本地临时副本。

## 工具选择原则

- 首选 B 站公开接口读取标题、UP 主、发布时间、时长、`cid`、字幕列表和播放地址。
- 公开接口只能返回试播内容、缺失字幕或需要会员权限时，在用户明确授权后可使用 Chrome 中现有的 B 站登录态继续获取；即使已获登录态授权，也应优先尝试接口方式读取元数据、字幕和播放地址，仅在接口需要登录凭证或页面核验时使用浏览器登录会话。
- 首选视频自带字幕；字幕缺失或质量不足时使用 `faster-whisper` 转写。
- 使用 PyAV 精确定位时间点，Pillow 输出 JPEG 和候选帧联系表，再人工复核图片。
- 使用 `rg`、`sed` 检索转写稿，先建立观点时间轴和全部股票提及清单，再整理并冻结文字稿，最后生成总结。
- 股票名按“清晰画面 > 官方元数据或字幕 > 上下文 > 语音识别”核实；题材名及其他内容不做独立核实，按文字稿原始语境保留。
- 使用 `quick_validate.py`、站点构建命令和 `git diff --check` 做发布前检查。
- 原视频、音频和未经整理的自动转写只允许暂存在临时目录，不提交到仓库。整理后的可读时间轴文字稿必须写入本项目，任务结束前逐个清除本期全部临时文件。

## 文字稿规范

- 文字稿制作分为两个连续步骤，不得跳过原始提取稿直接写结构化文字稿或总结：
  1. **原始提取稿**：直接落下字幕或语音识别结果，尽量保持作者原始顺序、说法、语气强弱、重复、犹豫、自我修正、类比和例子，不先概括、改写、重组、核名替换或补充分析。仅可补充时间点、说话人和无法识别标记。
  2. **整理文字稿**：必须在原始提取稿上逐段处理，划分自然段和议题，做不改变原意与语气强弱的轻度文字润色，删除纯填充语气词和无意义重复，并结合画面与上下文修订明显识别错误；外部核实只用于确认股票名称，不核实题材名或其他内容。
- 原始提取稿是过程性底稿，可暂存在临时目录，不要求提交；项目中的视频文字稿为第二步完成的整理稿。
- 覆盖从开场到收尾的全部有效议题，不只摘录高光、结论或代表性股票。
- 每个自然议题标注起始时间，行情切换、板块切换、个股切换和观点转折处必须增加时间点。
- 整理时不得把口语改写成摘要或分析结论；最大化保留作者的论述顺序、常用说法、语气强弱、犹豫、自我修正、类比、例子和有信息量的重复，不能确定是否有信息量时予以保留。
- 作者对某一观点讲述较多、反复补充或连续推演时，整理文字稿必须给予与原视频相称的篇幅，完整保留其问题背景、判断依据、推理过程、举例、反例、条件限制、观点转折和最终落点。不得因观点可以概括而大幅缩写，也不得只留下首句和结论。
- 长观点可在同一时间轴议题下按作者的展开层次拆成多个自然段或小点，以提高可读性；拆分只能整理结构，不能合并、重排或删减关键论述。
- 自动转写不能直接标成逐字原话。明显识别错误按画面和上下文修正，无法确认的股票或专有名词保留时间点并标记“待核”。
- 单独维护“本期全部股票提及表”，记录股票名、代码（能确认时）、所属板块、出现时间、作者说法和核验状态。
- 同一股票多次出现时，不只保留第一次；应合并列出关键时间点和观点变化。
- 开头注明视频信息、转写来源、视频时长和准确性限制，并链接回原视频。
- 整理文字稿完成后，必须逐项核实“本期全部股票提及表”中的股票名；这是内容核实的唯一范围。题材逻辑、市场数据、主播判断和新闻背景不做额外事实核验。
- 只有股票名核实完成、覆盖度检查通过并写入正式文件的整理文字稿才算“冻结”。截图选择和总结必须以冻结文字稿为依据；不得在此之前创建或预写总结正文。

## 分析准则

- 将预测改写为条件句，例如“若次日放量站稳压力位，反转概率才提高”。
- 区分板块“涨得多”和“有持续性”。持续性至少考察逆势抗跌、修复主动、核心带动、量价健康。
- 区分消息刺激和内生趋势。消息日的一致高潮不等于可追买。
- 记录否证条件。没有退出条件的看多观点不算完整交易计划。
- 对个股仅复述视频语境，不提供确定性买卖指令。
- 普通复盘的板块速览表至少包含“板块、制作者观点、全部提及个股、次日关注”四列；会员复盘按 [references/member-video.md](references/member-video.md) 的精简模板执行。
- 视频明确提及的股票必须全部进入文字稿和“全部股票提及表”，不能因为不是核心、只出现一次或观点偏负面而省略。
- 普通复盘总结正文也要按板块保留全部提及股票；会员复盘笔记只保留影响判断的核心、竞争者和教学案例票，完整股票记录由会员文字稿承载。
- 股票名无法确认时保留其时间点、上下文和“待核”标记，不凭发音猜测，也不因不确定而静默删除。
- 总结优先忠实呈现作者的原始表达逻辑。结构化改写不得改变语气强弱，例如不能把“感觉可以看看”改成“明确看好”。
- 总结的详略应参考作者在视频中的实际论述篇幅和强调程度。作者展开较多的观点必须在总结中充分保留其依据、推演链条、例子、条件、风险和结论，不得为了控制篇幅压缩成速览表中的一句话或一条笼统结论。
- 对篇幅较长或层次较多的观点，应在“市场总览”“具体分析”“次日计划”或“知识点”等对应大模块中单独设置清晰的小点；即使速览表已经概括，也必须在正文继续展开。一个长观点涉及多个不同判断时，可拆成多个小点逐一保留。
- 对每个主要板块说明其定位：主线候选、消息修复、高低切、量化轮动、情绪 PK、避险或回避。
- 如转写与画面冲突，以画面和上下文为准；仍不确定时使用“疑似”或省略名称。

### 人物主页

只维护模块入口和说明，不堆放日期归档或单期正文。默认入口包括“交易日总结”“视频文字稿”“充电会员主题”“技术点学习”。充电会员主题页再维护“复盘合集”和“复盘文字稿”等子入口；未来遇到其他充电合集时，在 `member-topics/` 下按主题增设子目录。

### 交易日总结模块

在 `trading-days/index.md` 按日期倒序维护文章入口。每篇日期文章固定包含：

- `## 每日信息`
  - `### 第一部分：市场总览`
  - `### 第二部分：具体分析`
  - `### 第三部分：次日计划`
- `## 知识点`：当日方法提炼、适用条件、常见误区、失效条件和交易准则，明确标注分析与延展。

固定一级、二级结构不限制正文展开。凡作者在视频中重点长篇讨论的内容，可在对应部分下增加四级标题、加粗小标题或多个独立小点，篇幅以完整保留观点为优先，不以追求短摘要为目标。

### 文字稿模块

在本项目的 `docs/trading/experts/bibabu-zuoshou/transcripts/` 保存文字稿，并在 `transcripts/index.md` 按日期倒序维护入口。每篇文字稿包含视频信息、准确性说明、全片时间轴和“本期全部股票提及表”。文字稿服务于追溯作者完整表达，不替代原视频，也不把自动识别文本冒充逐字稿。

`trading-days/index.md` 的模块标题固定为“交易日总结”，`transcripts/index.md` 的模块标题固定为“视频文字稿”；日期文章标题分别使用 `YYYY-MM-DD 交易日总结` 和 `YYYY-MM-DD 视频文字稿`。

### 站点目录同步

每期文档落盘后必须将文章加入 `docs/.vitepress/config.mts` 的“比巴卜作手”侧边栏目录，并与模块索引保持一致：

- 普通复盘同时加入“交易日总结”和“视频文字稿”的日期列表。
- 会员复盘同时加入对应会员笔记与会员文字稿列表。
- 会员专题课程同时加入课程学习笔记与课程文字稿列表，并按课程编号排序。
- 日期纠正、文件改名或路径迁移时，必须同步替换旧目录链接，不能保留失效入口。
- 发布前使用 `rg` 对照文章路径、模块索引和 VitePress 配置，确认三处均已收录且日期倒序正确；站点构建成功不代表目录已经更新，必须单独核对。

### 充电会员主题模块

充电会员内容统一放入 `member-topics/`。目前已识别的子主题包括：

- `member-topics/reviews/YYYY-MM-DD-BVID.md`：充电会员复盘笔记。
- `member-topics/review-transcripts/YYYY-MM-DD-BVID.md`：充电会员复盘文字稿。
- `member-topics/hot-money-system/NN-course-slug.md`：游资交易系统课程学习笔记。
- `member-topics/hot-money-system-transcripts/NN-course-slug.md`：游资交易系统课程文字稿。

会员复盘目录标题使用“充电会员复盘”，会员复盘文字稿目录标题使用“充电会员复盘文字稿”。同一天出现多期同类型内容时必须保留 BVID，避免覆盖。

“游资交易系统”属于按学习顺序组织的体系课程，文件名使用课程编号，不使用发布日期作为主要排序依据。课程原始内容和忠实整理放在会员主题下；`technical-notes/` 仅用于跨期验证后的二次归纳，不直接承接单课内容。

会员复盘采用“文字稿负责完整，复盘笔记负责选择”的分工。复盘笔记固定使用“本期结论与交易剧本、盘面结构与重点方向、关键知识点与案例、次日观察清单”四个主体模块，具体筛选标准与模板见 [references/member-video.md](references/member-video.md)。后续迭代重心不是增加目录，而是减少重复、提高关键知识点和案例的提取质量；内部框架改进记录不写入正式复盘正文。
