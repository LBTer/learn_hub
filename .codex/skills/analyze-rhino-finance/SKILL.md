# 视野环球财经视频分析

## 执行前必读

处理视频前读取 [references/video-analysis-workflow.md](references/video-analysis-workflow.md)，按“元数据确认 -> 文字稿提取与整理 -> 全部标的核名 -> 文字稿冻结 -> 复盘生成 -> 目录更新 -> 校验与清理”的顺序执行。

执行过程中涉及 Python 脚本、`yt-dlp`、音频下载、语音转写、截图、行情核名或校验命令时，优先使用本项目本地 `uv` 环境运行，例如 `uv run ...`；只有本地 `uv` 环境缺少能力或命令失败且确认无法修复时，才临时改用系统环境，并在最终说明中标注原因。

## 类型识别与路由

1. 获取标题、频道、发布日期、时长、原始链接、`availability`、`live_status`、标签、简介、字幕、章节和会员限制信息。账号应为“视野环球财经 / @RhinoFinance”，频道 ID 可用作辅助核验。
2. 按以下证据识别类型，证据冲突时以 YouTube 可见性和实际内容结构为准：
   - **普通美股盘后复盘**：公开视频，围绕美股大盘、板块 ETF、核心个股、交易计划和风险条件展开，通常每个交易日盘后更新。
   - **普通专题视频**：公开视频，但聚焦单家公司、行业、ETF、投资心理或交易方法。
   - **会员研究视频**：`availability` 为 `subscriber_only` 或 `premium_only`，或页面要求加入频道会员；内容涉及个股/行业研究、短中长期机会、策略、心理、交易技巧或优先答疑。
   - **会员答疑/帖子延伸**：会员内容中以观众问题、个股估值、支撑压力、被套处理、板块机会为主。
3. 会员内容读取 [references/member-video.md](references/member-video.md)，生成会员文字稿和会员笔记。
4. 普通公开视频继续执行下方“普通视频工作流”。
5. 不得把普通盘后复盘写入会员目录；不得把会员研究硬套成普通盘后复盘。

## 普通视频工作流

1. 核验视频标题、频道、发布日期、时长和原始链接。日期以 YouTube 发布日期为准；若视频明确对应前一交易日，在文档中同时注明“发布日期”和“复盘交易日”。
2. 必须严格串行执行以下阶段，不得交叉生成文字稿和复盘：
   - **阶段一：文字稿提取与整理**。优先获取 YouTube 字幕；无字幕时下载音频并本地转写。先形成原始提取稿，再以原稿为底稿制作覆盖全片的可读时间轴文字稿。
   - **阶段二：全部标的核名**。读取 [references/ticker-verification.md](references/ticker-verification.md)，核实视频提到的全部股票、ETF、指数和重要资产代码。
   - **阶段门：文字稿冻结**。确认全片有效议题、全部标的、长篇观点、操作条件、风险条件和后续观察均已纳入，且“本期全部标的提及表”已完成。将文字稿完整写入正式文件后冻结。
   - **阶段三：美股盘后复盘**。只能在文字稿冻结后创建或填写复盘，并以冻结文字稿为唯一内容底稿。
   - **阶段四：发布检查**。完成索引、人物主页、侧边栏、必要构建校验和临时文件清理。
3. 普通公开视频每期必须产出两个文件：
   - `docs/trading/experts/rhino-finance/transcripts/YYYY-MM-DD.md`：覆盖全片议题的可读时间轴视频文字稿。
   - `docs/trading/experts/rhino-finance/market-reviews/YYYY-MM-DD.md`：基于完整文字稿制作的美股盘后复盘。
4. 若同一天多期同类型视频，文件名使用 `YYYY-MM-DD-视频ID.md` 避免覆盖。
5. 在文字稿冻结前禁止创建或预写复盘正文。复盘不得直接基于标题、标签、零散字幕、音频片段、截图、记忆或简介生成。
6. 按以下顺序梳理普通盘后复盘内容：
   - 市场环境：SPY、QQQ、IWM、DIA、VIX、利率、美元、宏观数据、支撑压力、趋势线、均线和破位/修复条件。
   - 板块与 ETF：软件、半导体、AI、大科技、成长股、价值股、防御板块，以及视频提到的 IGV、SOXX、SMH、XLK、XLF 等。
   - 核心个股：逐一记录 MSFT、GOOG/GOOGL、NVDA、TSLA、PLTR、AMD、AVGO、META、AMZN、AAPL 等实际提及标的。
   - 交易计划：观察条件、进场条件、减仓条件、避险条件、仓位和“不追高/不抄底/等待确认”等纪律。
   - 投资心理与方法：主播明确表达的风险控制、被套处理、趋势判断、支撑压力、短中长期区分。
   - 后续验证：下一个交易日或下一阶段需要观察的价格、板块强弱、资金流向和否证条件。
7. 普通专题视频仍保存到 `market-reviews/` 与 `transcripts/`，但复盘正文按实际主题调整，并在 `company-notes/` 或 `analysis-framework/` 只沉淀跨期归纳，不把单期内容直接搬成长期结论。
8. 涉及具体公司、ETF、指数和宏观事实时，区分“可验证事实”“Rhino 观点”“分析者延展”。外部核验只用于确认标的身份、事实背景和明显识别错误，不得改写主播原意。
9. 必须保留原视频链接、配套视频文字稿链接、全部标的表、核验说明和“仅供学习，不构成投资建议”声明。
10. 同步更新所有目录入口：
   - 人物主页 `docs/trading/experts/rhino-finance/index.md` 的最新一期。
   - 美股盘后复盘索引 `docs/trading/experts/rhino-finance/market-reviews/index.md`。
   - 视频文字稿索引 `docs/trading/experts/rhino-finance/transcripts/index.md`。
   - VitePress 侧边栏 `docs/.vitepress/config.mts` 中“视野环球财经”的“美股盘后复盘”和“视频文字稿”条目。
11. 全部正式产物生成并检查完成后，逐个删除本次任务生成且不再需要的下载、字幕、音频、转写中间稿、候选帧和其他过程文件。

## 文字稿规范

- 文字稿制作分为两个连续步骤：
  1. **原始提取稿**：直接落下字幕或语音识别结果，尽量保持主播原始顺序、说法、语气、重复、自我修正、例子和条件句，不先概括、重组或补充分析。
  2. **整理文字稿**：必须在原始提取稿上逐段处理，划分自然段和议题，做不改变原意的轻度文字润色，并结合画面、标题、标签、简介和可靠来源修订明显 ticker 或专有名词错误。
- 覆盖从开场到收尾的全部有效议题，不只摘录标题相关内容。
- 每个自然议题标注起始时间；市场切换、ETF 切换、个股切换、观点转折和交易条件变化处必须增加时间点。
- 不得把口语改写成摘要或确定性买卖建议。主播用“可能”“观察”“略硬”“辛苦”“风险未解除”等语气强弱时，整理稿和复盘都要保留判断强度。
- 单独维护“本期全部标的提及表”，记录 ticker、名称、类型、出现时间、主播说法、核验状态和备注。
- 同一 ticker 多次出现时，不只保留第一次；应合并列出关键时间点和观点变化。
- 开头注明视频信息、转写来源、视频时长、准确性限制和原视频链接。
- 只有标的核名完成、覆盖度检查通过并写入正式文件的整理文字稿才算“冻结”。

## 分析准则

- 严格区分可验证事实、主播观点和分析者延展。
- 标题不是结论；必须回到正文论证、图表和条件。
- 将预测整理为条件句，保留时间尺度和否证条件。
- 对个股仅复述视频语境，不提供确定性买卖指令。
- 区分短线交易、中线布局和长期投资，不把一个时间尺度的观点外推到另一个时间尺度。
- 区分“强于大盘”“已经反转”“只是横盘抗跌”“仍在破位风险中”等不同判断层级。
- 对 ETF 和个股分开记录，不能把 ETF 表现直接等同于单家公司基本面。
- 复盘详略参考主播实际讲述篇幅和强调程度。长观点必须保留背景、依据、推演链条、例子、条件、风险和结论。
- 若视频没有字幕或转写质量不足，不伪造主播原话、时间点或 ticker；先补录或标记“待补录/待核”。

## 临时文件清理

- 创建临时文件时同步记录明确路径，只清理本次任务生成的文件。
- 清理必须放在工作流最后：先确认正式文字稿、复盘、索引、人物主页和侧边栏均已写入并完成必要检查，再删除临时文件。
- 每次只能删除一个明确路径的文件；禁止使用通配符、目录递归删除或任何批量删除命令，包括 `rm -rf`、`rmdir`、`Remove-Item -Recurse`。
- 临时目录中含有多个文件时，逐个核对并逐个删除；不得直接删除整个目录。若必须批量删除，应停止并请求用户手动处理。
