# 袁帅马视频分析

## 执行前必读

处理视频前读取 [references/video-analysis-workflow.md](references/video-analysis-workflow.md)，按“元数据确认 -> 文字稿提取与整理 -> 股票名与画面核实 -> 文字稿冻结 -> 分析笔记生成 -> 目录更新 -> 校验与清理”的顺序执行。

执行过程中涉及 Python 脚本、下载、转写、截图、图片处理或校验命令时，优先使用本项目本地 `uv` 环境运行，例如 `uv run ...`；只有本地 `uv` 环境缺少能力或命令失败且确认无法修复时，才临时改用系统环境，并在最终说明中标注原因。

## 类型识别与路由

1. 获取标题、UP 主、发布日期、时长、原始链接、分 P、`cid`、字幕列表、页面可见性、充电限制、合集信息和简介。账号名以 B 站页面为准，当前观察对象为“袁帅马”。
2. 按实际内容识别类型，证据冲突时以页面可见性和视频结构为准：
   - **普通日常分析**：公开视频，围绕某只或少数几只个股的历史走势、趋势结构、主力操盘、筹码/量价/均线/形态、参与条件和风险边界展开；可包含若干后续可参与个股的简要分析。
   - **普通专题教学**：公开视频，不以某一期日常分析为主，而是集中讲趋势、主力行为、盘口、买卖点、仓位、心态或其他交易方法。
   - **充电会员合集视频**：页面或接口显示充电专属、会员可见，或属于充电合集；内容可能是单股深度讲解、趋势教学、案例课、日常分析加强版或问答。
   - **其他内容**：若出现直播切片、社群说明、纯公告等，先如实记录类型，再决定是否只生成文字稿。
3. 充电会员内容读取 [references/member-video.md](references/member-video.md)，必须先识别所属充电合集主题，再按“主题笔记目录 + 主题文字稿目录”的成对结构生成会员文字稿和会员笔记；遇到新合集时按合集主题建立子目录，再回头迭代工作流。
4. 普通日常分析和普通专题教学继续执行下方“普通公开视频工作流”。
5. 不得把个股趋势分析硬套成市场交易日复盘，也不得把会员合集内容写入普通日常分析目录。

## 普通公开视频工作流

1. 核验视频标题、UP 主、发布日期、时长和原始链接。日期以 B 站发布时间为准；若视频明确分析某个历史区间或交易日，在文档开头同时注明。
2. 必须严格串行执行以下阶段，不得交叉生成文字稿和分析笔记：
   - **阶段一：文字稿提取与整理**。优先获取字幕；无字幕时下载音频并本地转写。先形成原始提取稿，再以原稿为底稿制作覆盖全片的可读时间轴文字稿。
   - **阶段二：股票名与画面核实**。结合清晰画面、字幕、上下文和必要的公开行情信息，核实全部股票名、代码、图表时间、关键价格区间和作者指认的走势位置。
   - **阶段门：文字稿冻结**。确认全片有效议题、全部股票、历史走势解析、重点参与条件、风险边界、长篇观点和简要个股清单均已纳入，且“本期全部标的与图表信息表”已完成。将文字稿完整写入正式文件后冻结。
   - **阶段三：日常分析生成**。只能在文字稿冻结后创建或填写分析笔记，并以冻结文字稿为唯一内容底稿。
   - **阶段四：发布检查**。完成关键帧、索引、人物主页、侧边栏、必要构建校验和临时文件清理。
3. 普通公开视频每期必须产出两个文件：
   - `docs/trading/experts/yuanshuaima/transcripts/YYYY-MM-DD-BVID.md`：覆盖全片议题的可读时间轴视频文字稿。
   - `docs/trading/experts/yuanshuaima/daily-analysis/YYYY-MM-DD-BVID.md`：基于完整文字稿制作的日常分析笔记。
4. 在文字稿冻结前禁止创建或预写日常分析正文。分析笔记不得直接基于标题、截图、零散字幕、原始转写、记忆或外部行情生成。
5. 普通日常分析按以下顺序整理：
   - 本期对象：核心个股、辅助个股、分析区间、作者本期要解决的问题。
   - 历史走势：按作者讲述顺序保留阶段划分、关键转折、趋势结构、成交量、均线、筹码、形态、盘口或主力动作。
   - 主力行为与趋势判断：记录作者如何解释吸筹、洗盘、试盘、拉升、出货、突破、回踩、平台整理、震荡或趋势延续。
   - 后续参与：作者明确给出的观察位置、介入前提、仓位含义、止损/失效条件、等待信号和不应参与的位置。
   - 简要个股：作者抽取出来简单讲后续如何参与的股票，要保留股票名、出现时间、作者说法、参与条件和风险。
   - 方法提炼：只提炼本期视频明确讲出的趋势分析、主力行为识别或买卖点方法，不把单期案例直接扩张为稳定战法。
6. 选择足以覆盖核心个股完整推演和简要个股参与条件的关键截图，通常为 6 至 12 张。优先截取图表结构、关键位置、作者标注、买卖点、失效条件和简要个股清单，避免只截口播画面。
7. 保留原视频链接、项目内文字稿链接、全部标的与图表信息表、核验说明和“仅供学习，不构成投资建议”声明。
8. 同步更新人物主页、日常分析索引、视频文字稿索引和 `docs/.vitepress/config.mts` 中“袁帅马”侧边栏目录。新增、改名或迁移文章时，必须同步检查侧边栏入口。

## 文字稿规范

- 文字稿制作分为两个连续步骤：
  1. **原始提取稿**：直接落下字幕或语音识别结果，尽量保持作者原始顺序、说法、语气强弱、重复、自我修正、类比和例子，不先概括、重组、核名替换或补充分析。
  2. **整理文字稿**：必须在原始提取稿上逐段处理，划分自然段和议题，做不改变原意与判断强度的轻度润色，并结合画面与上下文修订明显识别错误。
- 覆盖从开场到收尾的全部有效议题，不只摘录核心结论或代表性股票。
- 每个自然议题标注起始时间；股票切换、图表区间切换、趋势阶段切换、观点转折、参与条件和风险条件变化处必须增加时间点。
- 整理时不得把口语改写成摘要或确定性买卖建议；作者使用“可以看看”“等回踩”“不追”“风险大”“条件满足才参与”等语气时，文字稿和笔记都要保留判断强度。
- 单独维护“本期全部标的与图表信息表”，记录股票名、代码、类型、出现时间、作者说法、画面证据、参与条件和核验状态。
- 同一股票多次出现时，不只保留第一次；应合并列出关键时间点、图表阶段和观点变化。
- 开头注明视频信息、转写来源、视频时长、准确性限制和原视频链接。
- 只有股票名和关键画面信息核实完成、覆盖度检查通过并写入正式文件的整理文字稿才算“冻结”。

## 分析准则

- 严格区分“视频可见信息”“作者观点”“分析者整理/延展”。
- 标题不是结论；必须回到视频正文、图表和作者推演。
- 对个股仅复述视频语境，不提供确定性买卖指令。
- 将参与计划整理为条件句，保留触发条件、等待条件、否证条件和风险边界。
- 不把单股案例直接总结成稳定战法；至少积累多期后再沉淀到 `technique-notes/`。
- 对作者讲得长、反复强调或作为教学重点的内容，要保留背景、图表依据、推演链条、例子、条件和结论。
- 图表画面优先于转写结果；股票名、代码、日期、价格区间和指标无法确认时标记“待核”。
- 若视频无字幕或转写质量不足，不伪造作者原话、时间点或股票名；先补录或标记“待补录/待核”。

## 临时文件清理

- 创建临时文件时同步记录明确路径，只清理本次任务生成的文件。
- 清理必须放在工作流最后：先确认正式文字稿、日常分析、会员笔记、索引、人物主页和侧边栏均已写入并完成必要检查，再删除临时文件。
- 每次只能删除一个明确路径的文件；禁止使用通配符、递归删除或批量删除目录。
- 临时目录中含有多个文件时，逐个核对并逐个删除；若必须批量删除，应停止并请求用户手动处理。
